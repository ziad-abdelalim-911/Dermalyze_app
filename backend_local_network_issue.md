# 🛑 حل مشكلة الاتصال بالسيرفر المحلي (Localhost) من تطبيق الموبايل

المشكلة دي بتحصل دايماً لما الباك إند بيكون شغال "Locally" على جهاز الكمبيوتر، وتطبيق الموبايل (سواء على Emulator أو موبايل حقيقي) بيحاول يكلمه عن طريق `localhost` أو `127.0.0.1`.

الموبايل مش بيفهم `localhost` على إنها كمبيوتر الباك إند، الموبايل بيفهم إن `localhost` دي هي **الموبايل نفسه**! عشان كده الـ Endpoints مش بتشتغل وبترجع Connection Error.

عشان نحل المشكلة دي، لازم الباك إند وتطبيق الموبايل يكونوا متوصلين ببعض عن طريق **الـ IP الفعلي** للكمبيوتر على شبكة الـ Wi-Fi.

---

## 👨‍💻 المطلوب من مطور الباك إند (Backend Developer)

عشان الموبايل يقدر يشوف السيرفر بتاعك، لازم تعمل خطوتين:

### 1. تشغيل السيرفر على شبكة الـ Wi-Fi
تأكد إن السيرفر بتاعك مش بيـ Listen على `localhost` بس. لازم تخليه يـ Listen على كل الـ IPs المتاحة `0.0.0.0`.
- **في Node.js / Express:**
  ```javascript
  // ❌ غلط (بيخلي السيرفر مقفول على الكمبيوتر بس)
  app.listen(3000, 'localhost', () => { ... });

  // ✅ صح (بيفتح السيرفر لأي جهاز على نفس شبكة الواي فاي)
  app.listen(3000, '0.0.0.0', () => { ... });
  ```

### 2. معرفة الـ IP بتاع الكمبيوتر
لازم تجيب الـ IPv4 Address بتاع جهازك وتبعتو لمطور الموبايل عشان يحطه في الكود عنده.
- **لو أنت Windows:** افتح الـ CMD واكتب `ipconfig` ودور على `IPv4 Address` (هتلاقيه حاجة زي `192.168.1.5`).
- **لو أنت Mac/Linux:** افتح الـ Terminal واكتب `ifconfig` أو `ip a`.

### 3. إيقاف الـ Firewall (لو لزم الأمر)
لو عملت الخطوات اللي فوق والموبايل لسه مش عارف يوصل، تأكد إن الـ Windows Defender Firewall مش بيعمل Block للـ Port بتاعك (مثلاً بورت 3000). ممكن تقفل الفايروول مؤقتاً للتجربة.

## 📱 المطلوب من مطور الموبايل (Flutter Developer)

لازم تغير الـ `baseUrl` في كود الفلاتر عشان يشاور على كمبيوتر الباك إند صح:

### حالة 1: لو بتجرب على موبايل حقيقي (Real Device)
لازم الموبايل واللاب توب يكونوا متوصلين على **نفس شبكة الـ Wi-Fi**.
هتغير الـ URL في ملف `api_constants.dart` أو `dio_client.dart` للـ IP اللي الباك إند بعتهولك:
```dart
// ❌ غلط
final String baseUrl = "http://localhost:3000/api";
final String baseUrl = "http://127.0.0.1:3000/api";

// ✅ صح (استخدم الـ IP بتاع اللاب توب)
final String baseUrl = "http://192.168.1.5:3000/api"; 
```

### حالة 2: لو بتجرب على محاكي أندرويد (Android Emulator)
محاكي الأندرويد ليه IP مخصوص بيشاور بيه على الكمبيوتر اللي مشغله، وهو `10.0.2.2`.
```dart
// ✅ صح (في حالة الـ Emulator فقط)
final String baseUrl = "http://10.0.2.2:3000/api";
```

### حالة 3: لو بتجرب على محاكي آيفون (iOS Simulator)
محاكي الآيفون بيتعامل كأنه جزء من الكمبيوتر، فممكن يشتغل مع `localhost` عادي، بس الأفضل دايماً تستخدم الـ IP الفعلي `192.168.x.x` عشان تضمن إن مفيش مشاكل.
